Zertifikatsanzeige

Die Zertifikatsanzeige enthält Informationen über ein Zertifikat, das Sie auf einer der Registerkarten des Zertifikats-Managers ausgewählt haben. Die Registerkarte Allgemein enthält alle Informationen zum Aussteller des Zertifikats, seinem Auswertungsstatus sowie Verwendungszweck und Ähnliches. Sämtliche Details zum Inhalt des Zertifikats finden Sie auf der Registerkarte Details.

Wenn Sie die Zertifikatsanzeige noch nicht geöffnet haben, gehen Sie folgendermaßen vor:

  1. Klicken Sie im Menü Bearbeiten auf Einstellungen.
  2. Klicken Sie unter der Kategorie Privatsphäre & Sicherheit auf Zertifikate. (Wenn in dieser Kategorie keine Einträge angezeigt werden, doppelklicken Sie auf Privatsphäre & Sicherheit, um die Liste zu erweitern.)
  3. Klicken Sie auf Zertifikate verwalten.
  4. Klicken Sie auf die Registerkarte für den Zertifikattyp, dessen Details Sie anzeigen möchten.
  5. Wählen Sie das Zertifikat aus, dessen Details Sie anzeigen möchten.
  6. Klicken Sie auf Anzeigen.

Registerkarte Allgemein

Wenn Sie die Zertifikatsanzeige öffnen, wird die Registerkarte Allgemein mit verschiedenen Informationen über das ausgewählte Zertifikat angezeigt.

  • Dieses Zertifikat wurde für folgende Zwecke zugelassen: Im Abschnitt Zertifikatsüberprüfung finden Sie Erläuterungen dazu, wie der Zertifikats-Manager Zertifikate überprüft. Zu den Verwendungszwecken gehören:
    • SSL-Client-Zertifikat: Zertifikat zur Identifikation gegenüber Websites.
    • SSL-Server-Zertifikat: Zertifikat zur Identifikation eines Website-Servers gegenüber Browsern.
    • Zertifikat für eMail-Unterzeichner: Zertifikat, das Sie zum Zweck der digitalen Unterzeichnung von eMail-Nachrichten identifiziert.
    • Zertifikat für eMail-Empfänger Zertifikat zur Identifikation einer anderen Person, beispielsweise um dieser Person verschlüsselte eMail-Nachrichten senden zu können.
    • Zertifikat für Statusrückmeldung: Zertifikat zur Identifikation einer Online-Statusrückmeldung, die das Online Certificate Status Protocol (OCSP) zur Überprüfung der Gültigkeit von Zertifikaten verwendet. Weitere Informationen zu OCSP finden Sie unter Auswertungseinstellungen.
    • SSL-Zertifikatsaussteller: Zertifikat zur Identifikation eines Zertifikatsausstellers, d. h. eines Dienstes, der Zertifikate zur Identifikation über Computernetzwerke ausstellt.
  • Ausgestellt für: Fasst folgende Informationen über das Zertifikat zusammen:
    • Gemeinsamer Name: Name der Person oder einer anderen Entität, die durch das Zertifikat identifiziert wird.
    • Organisation: Name der Organisation, zu der die Entität gehört (z. B. der Name einer Firma).
    • Organisationseinheit: Name der Organisationseinheit, zu der die Entität gehört (z. B. Buchhaltungsabteilung).
    • Seriennummer: Die Seriennummer des Zertifikats.
  • Ausgestellt von: Fasst Informationen (ähnlich den unter "Ausgestellt für" bereitgestellten Informationen; siehe oben) über den Aussteller des Zertifikats (ZA) zusammen.
  • Gültigkeit: Gibt den Gültigkeitszeitraum des Zertifikats an.
  • Fingerabdrücke: Listet die Fingerabdrücke des Zertifikats auf. Ein Fingerabdruck ist eine eindeutige Nummer, die durch Anwendung einer mathematischen Funktion auf den Inhalt des Zertifikats erzeugt wird. Mithilfe eines Fingerabdrucks eines Zertifikats lässt sich feststellen, ob das Zertifikat manipuliert wurde.

 

Registerkarte Details

Auf der Registerkarte Details in der Zertifikatsanzeige finden Sie ausführlichere Informationen über das ausgewählte Zertifikat. Um Informationen zu einem Zertifikat aus dem Bereich Zertifikatshierarchie einzusehen, wählen Sie unter Zertifikatsfelder das Feld aus, das Sie untersuchen möchten. Unter Feldwert können Sie dann den entsprechenden Wert einsehen.

  • Zertifikatshierarchie: Zeigt die Zertifikatskette, an deren Ende sich das ursprünglich ausgewählte Zertifikat befindet. Eine Zertifikatskette ist eine hierarchische Abfolge von Zertifikaten, die von aufeinanderfolgenden Zertifikatsausstellern (ZAs) unterzeichnet sind. Ein ZA-Zertifikat identifiziert einen Zertifikatsaussteller und wird verwendet, um die von diesem Aussteller ausgegebenen Zertifikate zu unterzeichnen. Ein ZA-Zertifikat kann wiederum vom ZA-Zertifikat eines übergeordneten Ausstellers bis hin zum Stamm-ZA unterzeichnet sein.
  • Zertifikatsfelder: Zeigt die Felder des unter Zertifikatshierarchie ausgewählten Zertifikats an.
  • Feldwert: Zeigt den Wert des unter Zertifikatsfelder ausgewählten Feldes an.

In der Zertifikatsanzeige werden grundlegende mögliche ANSI-Typen in lesbarer Form angezeigt. Für Felder, deren Inhalt der Zertifikats-Manager nicht interpretieren kann, werden die tatsächlich im Zertifikat enthaltenen Werte angezeigt.

 

Auswählen von Sicherheitseinrichtungen

Eine Sicherheitseinrichtung (auch als "Token" bezeichnet) ist eine Hardware- oder Software-Einrichtung, die kryptografische Dienste wie Verschlüsselung und Entschlüsselung bereitstellt und Zertifikate und Schlüssel speichert. Das Fenster Sicherheitseinrichtung auswählen wird geöffnet, wenn der Zertifikats-Manager nicht selbst entscheiden kann, welche Sicherheitseinrichtung beim Importieren eines Zertifikats oder bei der Durchführung einer kryptografischen Operation, beispielsweise beim Erzeugen eines Schlüssels für ein neues Zertifikat, zu verwenden ist. In diesem Fenster können Sie eine der vom Zertifikats-Manager auf Ihrem Computer gefundenen Sicherheitseinrichtungen auswählen.

Eine SmartCard ist eine Möglichkeit für eine Sicherheitseinrichtung. Beispiel: Wenn sich in einem an Ihren Computer angeschlossenen SmartCard-Leser eine SmartCard befindet, wird der Name dieser Karte im Dropdown-Menü angezeigt. In diesem Falle müssen Sie den Namen der SmartCard auswählen, um dem Zertifikats-Manager mitzuteilen, dass Sie diese verwenden möchten.

Der Zertifikats-Manager stellt auch eine eigene eingebaute Sicherheitseinrichtung zur Verfügung, die immer verwendet werden kann, unabhängig von zusätzlich verfügbaren Sicherheitseinrichtungen.

 

Kopie des Kodierschlüssels

Zertifikatsaussteller (ZAs), die separate Zertifikate für die Unterzeichnung und Verschlüsselung von eMail ausstellen, legen während der Zertifikatsregistrierung Sicherheitskopien Ihres privaten Kodierschlüssels an.

Im Dialogfeld Kopie des Kodierschlüssels können Sie dem Erstellen der Sicherheitskopie zustimmen oder die Zertifikatsanforderung abbrechen. Ein ZA, der eine Sicherheitskopie Ihres Kodierschlüssels archiviert hat, könnte theoretisch jede ihrer eingehenden Nachrichten entschlüsseln, die mit ihrem entsprechenden öffentlichen Schlüssel kodiert wurde.

Im Dialogfeld Kopie des Kodierschlüssels können Sie Folgendes ausführen:

  • Zertifikat anzeigen: Wenn Sie das Zertifikat anzeigen möchten aus dem der ZA ersichtlich ist, klicken Sie auf Zertifikat anzeigen.

  • OK: Wenn Sie dem im Zertifikat angegebenen Aussteller die Entschlüsselung der von Ihnen empfangenen verschlüsselten Nachrichten anvertrauen möchten, klicken Sie auf OK.

    Wenn Sie nicht sicher sind, ob Sie dem ZA, der die Sicherheitskopie anfordert, vertrauen können, wenden Sie sich an Ihren Systemadministrator.

  • Abbrechen: Wenn Sie dem ZA, der eine Sicherheitskopie anfordert, nicht vertrauen, sollten Sie kein Zertifikat von ihm beantragen. Klicken Sie auf Abbrechen, um den Zertifikatsantrag zu verwerfen.

Wenn Ihr ZA eine Sicherheitskopie Ihres Kodierschlüssels angelegt hat, können Sie auf diese Kopie zurückgreifen, falls Sie Ihr Passwort oder Ihre eigene Kopie des Kodierschlüssels verlieren. Wenn keine Sicherheitskopie vorhanden ist, können Sie nach Verlust des Passworts oder des Schlüssels nicht mehr auf eMail-Nachrichten zugreifen, die mit diesem Schlüssel kodiert wurden.

 

Zertifikatssicherung

Wenn Sie ein Zertifikat erhalten haben, fertigen Sie eine Sicherheitskopie des Zertifikats und Ihres privaten Schlüssels an und bewahren Sie diese an einem sicheren Ort auf. Sie können die Kopie beispielsweise auf einer Diskette speichern und diese unter Verschluss nehmen. Auf diese Weise können Sie das Zertifikat problemlos wiederherstellen, falls Ihre Festplatte oder darauf befindliche Dateien beschädigt werden.

Je nach Verwendung kann der Verlust des Zertifikats und des dazugehörigen privaten Schlüssels im günstigsten Falle Unannehmlichkeiten bereiten, in einigen Fällen sogar katastrophale Auswirkungen haben. Beispiele:

  • Wenn Sie ein Zertifikat verlieren, das Sie gegenüber wichtigen Websites identifiziert, können Sie auf diese Websites erst wieder zugreifen, wenn Sie ein neues Zertifikat beantragt haben.
  • Wenn Sie ein Zertifikat zur Verschlüsselung von eMail-Nachrichten verlieren, können Sie verschlüsselte Nachrichten – sowohl gesendete als auch empfangene – nicht mehr lesen. Wenn Sie keine Sicherheitskopie des privaten Kodierschlüssels zur Verfügung haben, sind die mit diesem Schlüssel kodierten Nachrichten für immer verloren.

Wie andere wertvolle Daten sollten auch Zertifikate gesichert werden, um spätere Probleme und Kosten zu vermeiden. Um sicherzugehen, sollten Sie sofort eine Sicherheitskopie anlegen.

 

Benutzeridentifikationsanforderung

Einige Websites erfordern die Identifikation durch ein Zertifikat statt durch einen Benutzernamen und ein Passwort, da Zertifikate eine zuverlässigere Form der Identifikation darstellen. Diese Art der Identifikation wird auch als Client-Echtheitsbestätigung bezeichnet.

Der Zertifikats-Manager kann allerdings mehrere Zertifikate für die Identifikation gegenüber Websites speichern. In diesem Fall wird im Zertifikats-Manager das Dialogfeld Benutzeridentifikationsanforderung angezeigt. Es enthält zwei Arten von Informationen:

Diese Site fordert, dass Sie sich mit einem Zertifikat identifizieren: In diesem Bereich des Dialogfelds werden die folgenden Informationen aufgeführt:

  • Host-Name: Der Name des Servers, der die Identifikation anfordert. Er wird als Teil des URL verwendet. Beispielsweise lautet der Name der Website von Netscape home.netscape.com.
  • Organisation: Der Name der Organisation, die die Website führt.
  • Ausgestellt von: Der Name des Zertifikatsausstellers (ZA).

Wählen Sie ein Zertifikat für die Identifikation: Die verfügbaren Zertifikate, mit denen Sie sich gegenüber Websites identifizieren können, sind in diesem Bereich des Dialogfelds in der Dropdown-Liste enthalten. Wählen Sie das Zertifikat aus, das am wahrscheinlichsten von der Website, die Sie besuchen möchten, anerkannt wird.

Um Ihnen die Entscheidung zu erleichtern, werden die Details des ausgewählten Zertifikats angezeigt:

  • Ausgestellt für: Führt die Informationen zu der vom Zertifikat identifizierten Person (z. B. Name und eMail-Adresse) sowie die Seriennummer und den Gültigkeitszeitraum des Zertifikats auf.
  • Ausgestellt von: Fasst die Informationen über den Aussteller des Zertifikats (ZA) zusammen, beispielsweise Namen, Standort und Bundesland.

 

Neuer Zertifikatsaussteller

Zu den vom Zertifikats-Manager verwalteten Zertifikaten, die entweder auf Ihrem Computer oder auf einer externen Sicherheitseinrichtung wie einer SmartCard gespeichert sind, gehören Zertifikate, die Zertifikatsaussteller (ZAs) identifizieren. Um andere Zertifikate erkennen zu können, muss der Zertifikats-Manager Zertifikate für die ZAs haben, die diese Zertifikate ausgestellt oder deren Ausstellung autorisiert haben.

Wenn Sie sich entscheiden, einem ZA zu vertrauen, führt der Zertifikats-Manager einen Download des Zertifikats dieses ZA durch und kann dann die von diesem Aussteller stammenden Zertifikate erkennen.

Vor dem Downloaden eines neuen ZA-Zertifikats können Sie im Zertifikats-Manager gegebenenfalls den Grund angeben, weshalb Sie dem Zertifikat vertrauen. Sie können eine der folgenden Optionen auswählen:

  • Diesem ZA bei der Kennzeichnung von Websites vertrauen: Zertifikate für einige Websites, beispielsweise solche, die finanzielle Transaktionen durchführen, können sehr wichtig sein und eine unangemessene oder falsche Identifikation kann negative Auswirkungen haben.
  • Diesem ZA bei der Kennzeichnung von eMail-Benutzern vertrauen: Wenn Sie eMail-Benutzern vertrauliche Informationen in verschlüsselter Form senden möchten oder wenn die zuverlässige Identifikation von eMail-Benutzern aus anderen Gründen für Sie wichtig ist, sollten Sie die Verfahrensweisen des ZA bei der Auswahl möglicher Zertifikatsinhaber sorgfältig auf die Eignung für Ihre Zwecke prüfen, bevor Sie diese Option wählen.
  • Diesem ZA bei der Kennzeichnung von Software-Herstellern vertrauen: Wenn Sie diese Option auswählen, vertrauen Sie dem ZA bezüglich der Ausstellung von Zertifikaten, die die Herkunft von Java-Applets und JavaScripts bezeichnen, die speziellen Zugang zu Ihrem Computer anfordern, beispielsweise um Dateien zu ändern. Da solche Zugriffsrechte missbraucht werden können, um beispielsweise Daten auf Ihrer Festplatte zu vernichten, sollten Sie mit dieser Option sehr vorsichtig umgehen, es sei denn, dass Sie sicher sind, dem ZA in diesem Punkt vertrauen zu können.

Bevor Sie sich entschließen, einem neuen ZA zu vertrauen, vergewissern Sie sich, wer der Betreiber ist. Stellen Sie sicher, dass die Richtlinien und Verfahrensweisen des ZA der Art der ausgestellten Zertifikate angemessen sind. Wenn der ZA beispielsweise Zertifikate zur Identifikation von Websites ausstellt, die Sie für finanzielle Transaktionen verwenden, überprüfen Sie, ob der Grad der Sicherheit, die der ZA bietet, ausreichend ist.

  • Anzeigen: Klicken Sie auf diese Schaltfläche, um das ZA-Zertifikat anzuzeigen, das Sie gerade downloaden möchten. Wenn Sie das Zertifikat nicht mehr downloaden möchten, klicken Sie auf Abbrechen.

Von unbekanntem Aussteller zertifizierte Website

Viele Websites verwenden Zertifikate, um sich zu identifizieren, wenn Sie die Site besuchen. Wenn der Zertifikats-Manager den Zertifikatsaussteller (ZA), der das Zertifikat der Website ausgestellt hat, nicht erkennt, wird eine Warnmeldung angezeigt, die Ihnen ermöglicht, das neue Website-Zertifikat zu untersuchen und zu entscheiden, was geschehen soll.

  • Zertifikat anzeigen: Klicken Sie auf diese Schaltfläche, um das Zertifikat der Website anzuzeigen.
  • Dieses Zertifikat dauerhaft speichern: Wenn Sie sicher sind, dass das Zertifikat gültig ist, wählen Sie diese Option aus und klicken Sie dann auf Weiter. Der Zertifikats-Manager wird dieses Zertifikat dann bis zum Ablauf seiner Gültigkeit als legitime Identifikation anerkennen.

Wenn Sie die Warnmeldung schließen und mit Ihrer Arbeit fortfahren möchten, klicken Sie auf Weiter oder Abbrechen.

  • Falls Sie den Verdacht haben, dass dieser Website nicht zu trauen ist, klicken Sie auf Abbrechen. Wenn Sie diese Option auswählen, stellt der Zertifikats-Manager dieses Mal die Verbindung zu der Website nicht her und zeigt beim nächsten Besuch erneut dasselbe Fenster an.

    Der Abbruch empfiehlt sich beispielsweise, wenn Sie finanzielle Transaktionen auf der Website durchführen. In diesem Fall sollten Sie das Problem der Bank oder sonstigen Organisation melden, die die Site betreibt, und sich vergewissern, dass das Zertifikat der Site gültig ist, bevor Sie fortfahren.

  • Falls Sie die Website trotzdem anzeigen möchten, klicken Sie auf Weiter.

    • Wenn Sie Dieses Zertifikat dauerhaft speichern aktiviert haben, erkennt der Zertifikats-Manager dieses Zertifikat bis zu seinem Ablauf als legitime Identifikation an.

    Haben Sie Dieses Zertifikat dauerhaft speichern nicht aktiviert, erkennt der Zertifikats-Manager dieses Zertifikat nur während der aktuellen Sitzung an. Beim nächsten Besuch der Website wird dieselbe Warnmeldung erneut angezeigt.

Wenn Sie Dieses Zertifikat dauerhaft speichern auswählen, lösen Sie das Problem für dieses Website-Zertifikat, die Warnmeldung wird jedoch für alle anderen Websites angezeigt, deren Zertifikate von demselben ZA stammen.

Wenn der Zertifikats-Manager allen von einem bestimmten ZA ausgestellten Zertifikaten vertrauen soll, müssen Sie die Vertrauenseinstellungen für das entsprechende ZA-Zertifikat bearbeiten. So greifen Sie auf diese Einstellungen zu:

  1. Klicken Sie im Menü Bearbeiten auf Einstellungen.
  2. Klicken Sie unter der Kategorie Privatsphäre & Sicherheit auf Zertifikate. (Wenn in dieser Kategorie keine Einträge angezeigt werden, doppelklicken Sie auf Privatsphäre & Sicherheit, um die Liste zu erweitern.)
  3. Klicken Sie auf Zertifikate verwalten.
  4. Klicken Sie auf die Registerkarte Aussteller.
  5. Wählen Sie das ZA-Zertifikat aus, dessen Vertrauenseinstellungen Sie bearbeiten möchten.
  6. Klicken Sie auf die Schaltfläche Bearbeiten und wählen Sie entsprechende Vertrauenseinstellungen aus.

 

Abgelaufenes Server-Zertifikat

Wie Kreditkarten, Personalausweise und andere Formen der Identifikation ist auch ein Zertifikat nur für einen bestimmten Zeitraum gültig. Wenn ein Zertifikat abläuft, muss der Inhaber ein Neues beantragen.

Der Zertifikats-Manager warnt Sie, falls Sie versuchen auf eine Website zuzugreifen, deren Server-Zertifikat abgelaufen ist. Sie sollten zunächst prüfen, ob Datum und Uhrzeit auf Ihrem Computer korrekt angezeigt werden. Wenn die Uhr Ihres Computers auf ein Datum eingestellt ist, das nach dem Ablaufdatum des Zertifikats liegt, betrachtet der Zertifikats-Manager das Zertifikat als abgelaufen.

Wenn die Uhr Ihres Computers richtig eingestellt ist, müssen Sie sich entscheiden, ob Sie der Site vertrauen möchten. Diese Entscheidung hängt davon ab, was Sie in der Site tun möchten und was Sie über die Site wissen. Die meisten kommerziellen Websites sorgen für einen rechtzeitigen Ersatz ihrer Zertifikate, bevor diese abgelaufen sind.

Im Dialogfeld Server-Zertifikat abgelaufen können Sie Folgendes ausführen:

  • Zertifikat anzeigen: Klicken Sie auf die Schaltfläche Zertifikat anzeigen, um Informationen über das Zertifikat, einschließlich Gültigkeitsdauer, einzusehen.

  • OK: Wenn Sie Grund zu der Annahme haben, dass es sich bei dem abgelaufenen Zertifikat um einen versehentlichen Fehler handelt, können Sie auf OK klicken und das Zertifikat für die aktuelle Sitzung akzeptieren. Informieren Sie den Webmaster der Site über das Problem.

    Seien Sie während des Besuchs der Site bei Ihren Aktionen vorsichtig.

  • Abbrechen: Wenn Sie den Verdacht haben, dass ein gravierendes Problem vorliegt und Sie den Besuch der Site generell nicht riskieren möchten, klicken Sie auf Abbrechen. Der Zertifikats-Manager stellt dann keine Verbindung zur Site her.

 

Server-Zertifikat noch nicht gültig

Wie Kreditkarten, Personalausweise und andere Formen der Identifikation ist auch ein Zertifikat nur für einen bestimmten Zeitraum gültig.

Der Zertifikats-Manager warnt Sie, falls Sie versuchen auf eine Website zuzugreifen, deren Gültigkeitszeitraum für das Server-Zertifikat noch nicht begonnen hat. Sie sollten zunächst prüfen, ob Datum und Uhrzeit auf Ihrem Computer korrekt angezeigt werden. Wenn die Uhr Ihres Computers auf das falsche Datum eingestellt ist, betrachtet der Zertifikats-Manager das Server-Zertifikat als noch nicht gültig, selbst wenn dies nicht der Fall ist.

Wenn die Uhr Ihres Computers richtig eingestellt ist, müssen Sie sich entscheiden, ob Sie der Site vertrauen möchten. Diese Entscheidung hängt davon ab, was Sie in der Site tun möchten und was Sie über sie wissen. Die meisten kommerziellen Websites sorgen dafür, dass ihre Zertifikate gültig sind, bevor sie diese in Gebrauch nehmen.

Im Dialogfeld Server-Zertifikat noch nicht gültig können Sie Folgendes ausführen:

  • Zertifikat anzeigen: Klicken Sie auf die Schaltfläche Zertifikat anzeigen, um Informationen über das Zertifikat, einschließlich Gültigkeitsdauer, einzusehen.

  • OK: Wenn Sie Grund zu der Annahme haben, dass es sich um einen versehentlichen Fehler handelt, können Sie auf OK klicken und das Zertifikat für die aktuelle Sitzung akzeptieren. Informieren Sie den Webmaster der Site über das Problem.

    Seien Sie während des Besuchs der Site bei Ihren Aktionen vorsichtig.

  • Abbrechen: Wenn Sie den Verdacht haben, dass ein gravierendes Problem vorliegt und Sie den Besuch der Site generell nicht riskieren möchten, klicken Sie auf Abbrechen. Der Zertifikats-Manager stellt dann keine Verbindung zur Site her.

 

Nicht übereinstimmender Domainname

Ein Server-Zertifikat gibt den Namen des Servers in Form des Domainnamens der Site an. Beispielsweise lautet der Domainname der Website von Netscape home.netscape.com. Wenn der Domainname in einem Server-Zertifikat nicht dem tatsächlichen Domainnamen der Website entspricht, kann das darauf hindeuten, dass jemand versucht, Ihre Kommunikation mit der Website abzufangen.

Die Entscheidung, ob Sie dieser Website dennoch vertrauen möchten, hängt davon ab, was Sie auf der Site zu tun beabsichtigen und was Sie über die Site wissen. Die meisten kommerziellen Websites sorgen dafür, dass der Host-Name des Website-Zertifikats dem tatsächlichen Host-Namen der Site entspricht.

Im Dialogfeld Domain-Namen stimmen nicht überein können Sie Folgendes ausführen:

  • Zertifikat anzeigen: Um die Informationen über das Zertifikat zu prüfen, klicken Sie auf Zertifikat anzeigen.

  • OK: Wenn Sie Grund zu der Annahme haben, dass es sich um einen versehentlichen Fehler handelt, können Sie auf OK klicken und das Zertifikat für die aktuelle Sitzung akzeptieren. Informieren Sie den Webmaster der Site über das Problem.

    Seien Sie während des Besuchs der Site bei Ihren Aktionen vorsichtig, und betrachten Sie die vorgefundenen Informationen mit Skepsis.

  • Abbrechen: Wenn Sie den Verdacht haben, dass ein gravierendes Problem vorliegt und Sie den Besuch der Site generell nicht riskieren möchten, klicken Sie auf Abbrechen. Der Zertifikats-Manager stellt dann keine Verbindung zur Site her.

Wenn Sie das Zertifikat dennoch für die aktuelle Sitzung akzeptieren, lassen Sie bei Ihrem Besuch Vorsicht walten und betrachten Sie die vorgefundenen Informationen mit Skepsis.
