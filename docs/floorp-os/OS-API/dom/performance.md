# パフォーマンスガイド

OS API のレイテンシ特性と最適化手法。エンドポイント選択からワークフロー設計まで。

---

## やりたいこと逆引き

| やりたいこと            | 推奨アプローチ                                   |
| ----------------------- | ------------------------------------------------ |
| Gmail 大規模 SPA を操作 | ax-tree + Text scoped + fingerprint の組み合わせ |
| 軽量サイトの全体取得    | Text full で一括取得、500 ms 以下で完了          |
| 大量タブを高速に作成    | `waitForLoad=false` で 31 ms/タブ                |
| クリック操作を高速化    | `stabilityTimeout=0` で安定性チェックをスキップ  |
| 部分更新を確認          | Text scoped で差分取得、全体再取得を避ける       |

---

## 基本フロー

効率的なページ操作の推奨手順:

```
1. GET /ax-tree          (0.20s)  構造把握・操作対象の特定
       ↓
2. POST /text (scoped)   (< 0.5s) 必要な範囲のテキスト + fingerprint 取得
       ↓
3. POST /click (fp)      (< 0.5s) fingerprint で要素をクリック
       ↓
4. GET /ax-tree          (0.20s)  操作結果の確認
```

合計 ~1.4 s で「構造把握 → テキスト取得 → 操作 → 確認」の 1 サイクルが完了する。

---

## 避けるべきパターン

| パターン                              | 問題                                                | 代替                         |
| ------------------------------------- | --------------------------------------------------- | ---------------------------- |
| 毎回 Text full を取得                 | Gmail で 0.60 s / 57 KB。SPA では大半が不変         | 初回のみ full、以降は scoped |
| HTML を取得してパース                 | 787 KB / 1.33 s。class 名が難読化されており解析困難 | ax-tree で構造把握           |
| CSS セレクタを推測して操作            | 難読化サイトで失敗する。試行錯誤でリクエスト増      | fingerprint を使う           |
| 毎回 screenshot を撮る                | 396 KB の画像転送。AI のトークン消費大              | 操作確認は ax-tree か exists |
| ページ遷移ごとに full page screenshot | 数百 KB × ページ数                                  | text で変化部分のみ確認      |

## DOM サイズとの相関

ページの DOM ノード数がエンドポイントの応答時間に与える影響:

| DOM ノード数 | text (ms) | ax-tree (ms) | html (ms) | 代表ページ        |
| ------------ | --------- | ------------ | --------- | ----------------- |
| ~100         | 5-15      | 5-15         | 5-10      | 静的な LP         |
| ~1,000       | 15-70     | 15-50        | 30-100    | 一般的なブログ    |
| ~5,000       | 70-250    | 50-150       | 100-400   | ECサイト商品一覧  |
| ~10,000      | 150-500   | 100-300      | 300-800   | SNS フィード      |
| ~30,000      | 500-1,000 | 200-500      | 800-2,000 | Gmail, 大規模 SPA |

text と html は DOM サイズにほぼ線形に比例する。ax-tree は不要ノードをフィルタするため増加が緩やか。

## 最適化オプション

### Text API

| オプション                  | デフォルト | 効果                                           |
| --------------------------- | ---------- | ---------------------------------------------- |
| `mode: "scoped"`            | `"full"`   | selector で範囲を限定しサイズと時間を大幅削減  |
| `mode: "visible"`           | `"full"`   | ビューポート内のみ取得。スクロール不要な確認に |
| `enableFingerprints: false` | `true`     | fingerprint を省略し出力サイズを約 15% 削減    |
| `includeSelectorMap: false` | `false`    | 対応表を省略 (デフォルトで省略済み)            |

### Click API

| オプション            | デフォルト | 効果                                                     |
| --------------------- | ---------- | -------------------------------------------------------- |
| `stabilityTimeout: 0` | (> 0)      | DOM 安定性チェックをスキップ。静的ページや連続クリックに |
| `force: true`         | `false`    | 表示状態チェックをスキップ。非表示要素のクリックに       |

### Instance API

| オプション           | デフォルト | 効果                                                 |
| -------------------- | ---------- | ---------------------------------------------------- |
| `waitForLoad: false` | `true`     | ページ読み込み完了を待たない。31 ms でタブ作成が返る |
| `inBackground: true` | `true`     | バックグラウンドでタブを開く。UI のちらつきを防止    |

## ワークフロー遅延予算

操作の種類別に目標レイテンシを示す。この予算内に収まるようエンドポイントとオプションを選択する。

### Gmail (大規模 SPA) — 添付ファイル操作の例

| ステップ                | エンドポイント | 所要時間   |
| ----------------------- | -------------- | ---------- |
| 1. メール一覧取得       | ax-tree        | 0.20 s     |
| 2. 対象メールのテキスト | text (scoped)  | 0.30 s     |
| 3. 添付ファイルクリック | click (fp)     | 0.32 s     |
| 4. ダウンロード確認     | exists         | 0.003 s    |
| 5. 結果確認             | ax-tree        | 0.20 s     |
| **合計**                |                | **~1.0 s** |

### 一般サイト — フォーム入力の例

| ステップ              | エンドポイント | 所要時間   |
| --------------------- | -------------- | ---------- |
| 1. フォーム構造取得   | ax-tree        | 0.05 s     |
| 2. フィールドに入力   | fill (fp)      | 0.05 s     |
| 3. 送信ボタンクリック | click (fp)     | 0.15 s     |
| 4. 結果確認           | text (scoped)  | 0.20 s     |
| **合計**              |                | **~0.5 s** |
